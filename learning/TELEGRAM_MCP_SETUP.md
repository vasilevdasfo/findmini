# 📱 Telegram MCP — настройка за 20 минут (а не за 3 дня)

> Дима на это убил 3 дня. Чтобы ты не повторял — вот короткая инструкция с подводными камнями.

---

## ❓ В чём вообще разница — Bot API vs MCP

Это разные вселенные, их путают все:

| | **Bot API** (@BotFather) | **Telegram MCP** (user client) |
|---|---|---|
| Кто действует | Бот с именем @xxx_bot | Твой личный аккаунт |
| Авторизация | Token от @BotFather | API_ID + API_HASH + phone + SMS |
| Может читать | Только чаты где бот добавлен | **ВСЕ твои чаты, группы, каналы** |
| Может писать | Только куда добавлен | От твоего имени куда угодно |
| Протокол | HTTP Bot API | MTProto (нативный Telegram) |
| Файл сессии | Не нужен | `.session` — береги как пароль |
| Claude Code юзает через | `curl` / `sendMessage` | MCP плагин `telegram-mcp` |

**Вывод:**
- `@demisworld_bot` (который у тебя в группе с Димой) — это **Bot API**. Пишет от имени бота. Читает только ту группу.
- `telegram-mcp` — это **твой аккаунт**. Claude может читать ВЕСЬ твой Telegram, отвечать клиентам от твоего имени, делать дайджесты переписок.

Тебе нужны **оба**. Но именно на MCP все спотыкаются.

## 🔑 Шаг 1. Получить API_ID + API_HASH (5 мин)

1. Открой https://my.telegram.org/auth
2. Логин: твой номер Telegram → придёт код в **Telegram (не SMS!)**
3. Меню **API development tools** → заполни форму:
   - App title: `claude-mcp` (любое)
   - Short name: `claudemcp`
   - Platform: `Desktop`
4. Сохрани **api_id** (число, типа `36335426`) и **api_hash** (32-символьная строка).

⚠️ **Подводный камень:** если форма «not available in your country» — VPN на Европу/США, открой снова.

---

## 🔧 Шаг 2. Конфиг Claude Code (2 мин)

Файл `~/.claude/mcp.json` — если его нет, создай:

```json
{
  "mcpServers": {
    "telegram": {
      "command": "npx",
      "args": ["-y", "telegram-mcp"],
      "env": {
        "TELEGRAM_API_ID": "ТВОЙ_API_ID",
        "TELEGRAM_API_HASH": "твой_api_hash_32_символа",
        "TELEGRAM_PHONE": "+79XXXXXXXXX"
      }
    }
  }
}
```

Номер — в международном формате, с плюсом, **тот самый что привязан к твоему Telegram**. Не левую симку.

Если у тебя уже есть другие MCP (perplexity и т.д.) — просто добавь секцию `"telegram"` внутрь `"mcpServers"`.

## 🚪 Шаг 3. Первая авторизация (где все ломаются)

1. Перезапусти Claude Code полностью (`Ctrl+C`, потом `claude` снова).
2. В чате напиши: `используй telegram mcp, покажи список моих чатов`.
3. Claude запустит MCP → в **терминале** (не в чате!) появится запрос:
   ```
   Please enter your phone code:
   ```
4. Код придёт **в официальное Telegram-приложение** на телефоне (сообщение от Telegram). **Не в SMS**. Вбей в терминал.
5. Если включена 2FA — попросит ещё и пароль.

Результат: создастся файл `.session` (обычно в рабочей директории) — это твой «вход» без повторного кода.

## ⚠️ 5 грабель, на которые наступил Дима

**1. «Код не приходит»**
Telegram шлёт код **в само приложение Telegram** (в «Telegram» от официального аккаунта), а не SMS. Открой приложение, жди 10–30 сек.

**2. «Phone number invalid»**
Номер без пробелов, скобок, дефисов. Только `+` и цифры. `+79091234567` — ОК. `+7 (909) 123-45-67` — фейл.

**3. «Session expired» через неделю**
Не удаляй `.session` файл. Если удалил — заново проходи авторизацию. Лежать должен там, откуда запускается Claude Code.

**4. MCP не видно в Claude**
Почти всегда — Claude Code не перезапущен после правки `mcp.json`. Полный выход (`Ctrl+C`, закрыть терминал), новый запуск.

**5. «Database is locked»**
Два процесса с одним `.session` — не работает. Закрой все параллельные `claude` / боты, использующие этот файл. Один пользователь = одна сессия одновременно.

## 🛡 Безопасность (прочитай один раз)

- `.session` файл = **полный доступ к твоему Telegram**. Кто его скопирует — читает твои переписки. Никогда не коммить в git, не клади в Dropbox.
- `api_hash` — тоже секрет. Не публикуй, не шарь.
- Claude может **отвечать людям от твоего имени**. Включай `permissions: ask` в Claude Code, пока не привыкнешь. Не давай `accept-edits` пока не поймёшь что MCP делает.
- Если телефон потерял/украли — зайди в my.telegram.org → **Active sessions** → terminate MCP-сессию.
