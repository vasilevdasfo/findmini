## 🎯 Принцип
Claude Code — это не «ещё один ChatGPT». Это **агент с руками**: читает файлы, правит код, пушит в git, управляет браузером, транскрибирует голос, шлёт в Telegram. Чем глубже настроен — тем больше делает сам. Цель: к 6-й неделе ты говоришь голосом, оно работает за тебя.

## L0 — База (день 0, 30–60 минут)

**Цель:** Claude Code запускается из терминала и понимает твой голос словами.

- [ ] **Подписка Claude.ai** — Pro ($20) минимум, лучше Max ($100–200). Без неё Claude Code не работает на полную.
- [ ] **macOS / Windows-WSL / Linux** — на чистом Windows не советую, половина инструментов не работает.
- [ ] **Установить Claude Code CLI:**
  ```bash
  curl -fsSL https://claude.ai/install.sh | bash
  # или через npm: npm install -g @anthropic-ai/claude-code
  ```
- [ ] **Войти:** `claude login` (откроет браузер, авторизация через claude.ai)
- [ ] **Первый запуск:** `cd ~ && claude` → поздоровайся, попроси `ls` — проверь что отвечает.
- [ ] **Git + GitHub аккаунт** — нужен для деплоя позже. Создай Personal Access Token (classic, с правами repo).
- [ ] **Редактор:** VS Code или Cursor (оба совместимы, Cursor глубже интегрирован).

**Чек:** в терминале `claude --version` выводит версию. Всё.

## L1 — Базовый workflow (неделя 1)

**Цель:** ты понимаешь, что такое контекст, permissions и CLAUDE.md. Делаешь простые задачи.

- [ ] **Команды:** `/help`, `/clear` (очистить контекст), `/init` (создать CLAUDE.md в проекте), `/config`, `/status`.
- [ ] **Permissions:** `Shift+Tab` переключает режимы (ask/accept/plan). Освой — это важно.
- [ ] **CLAUDE.md** в каждом проекте — это «память» для Claude. Положи туда:
  - Кто ты, что за проект, стек, стиль кода.
  - Команды деплоя, токены (без секретов), ссылки.
  - Что НЕ делать («не трогать папку X», «не коммитить без моей команды»).
- [ ] **Простые задачи:**
  - «Создай README.md с описанием проекта»
  - «Найди все TODO в коде и собери в один список»
  - «Почини ошибку в файле X строка Y»
- [ ] **Git:** попроси Claude закоммитить изменения. Обрати внимание — он пишет осмысленные сообщения, показывает diff перед пушем.
- [ ] **Esc** прерывает. `Ctrl+C` выходит. `@file` вставляет файл в контекст.

**Чек:** сделал реальную правку в любом проекте через Claude, закоммитил, запушил.

## L2 — MCP серверы (неделя 2)

**Цель:** Claude получает «руки» во внешние сервисы.

MCP (Model Context Protocol) — плагины. Конфиг: `~/.claude/mcp.json` или через `/mcp add`.

**Минимальный набор (в этом порядке):**
- [ ] **Perplexity MCP** — веб-поиск с источниками. Ключ: platform.perplexity.ai. ~$0.006/запрос.
- [ ] **Gmail MCP** — читать/писать черновики, искать треды. Через OAuth.
- [ ] **Google Calendar MCP** — смотреть расписание, создавать события.
- [ ] **Telegram MCP** (telegram-mcp) — читать/слать сообщения. Нужны API_ID и phone (my.telegram.org).
- [ ] **Claude Preview MCP** — запускать dev-сервер и проверять сайт браузером прямо из Claude.
- [ ] **Computer Use MCP** — управлять десктопом (скриншоты, клики). Использовать аккуратно.

**Проверка каждого:**
- «Найди в Gmail письма от X за неделю»
- «Что у меня в календаре завтра?»
- «Поищи в Perplexity: [вопрос]»

**Чек:** минимум 3 MCP работают, Claude их использует без подсказки.

## L3 — Skills и плагины (неделя 3)

**Цель:** Claude перестаёт задавать очевидные вопросы — сам знает как делать типовое.

- [ ] **Anthropic Skills** (через плагин marketplace): `docx`, `pdf`, `xlsx`, `pptx`, `perplex`, `skill-creator`, `youtube-transcript`.
- [ ] **Superpowers** плагин — для разработки:
  - `brainstorming` перед большой задачей
  - `writing-plans` перед реализацией
  - `systematic-debugging` при багах
  - `verification-before-completion` перед закрытием
- [ ] **Impeccable** плагин — для UI/дизайна: `polish`, `typeset`, `arrange`, `audit`, `animate`.
- [ ] **Создай первый свой skill** через `skill-creator`:
  - Пример: `/morning-brief` → читает Gmail + календарь + Telegram → даёт утреннюю сводку.
- [ ] **Slash-команды:** делай на частые операции. Примеры от Димы: `/boardroom`, `/meeting-prep`, `/content-machine`, `/crypto-dd`.

**Чек:** у тебя 1–2 своих skill'а, которые ты реально используешь ежедневно.

## L4 — Телеграм-боты и голос (неделя 4)

**Цель:** ты говоришь в телефон, Claude разбирает и кладёт в задачи/файлы.

- [ ] **ffmpeg** установлен (`brew install ffmpeg`).
- [ ] **Groq API** ключ (console.groq.com) — Whisper `whisper-large-v3` бесплатно, быстро.
  - Альтернатива: **Deepgram Nova-2** (платно, с диаризацией — кто говорит).
- [ ] **Свой Telegram-бот** через @BotFather → токен.
- [ ] **Python 3.11+** и `python-telegram-bot`, `groq`, `anthropic` библиотеки.
- [ ] **Первый бот:** голосовое → Whisper → текст → сохранить в `INBOX.md` проекта.
- [ ] **Второй бот:** текст → Claude (с твоим CLAUDE.md) → ответ.
- [ ] **launchd / systemd** — автозапуск ботов при старте мака.

Референс: `04_CYBOS/demi_tasks_bot.py` у Димы — рабочий пример.

**Чек:** ты диктуешь голосовое, через 20 секунд идея лежит текстом в файле проекта.

## L5 — Deploy pipeline (неделя 5)

**Цель:** сайт от идеи до продакшена за 10 минут голосом.

- [ ] **GitHub** — репо под каждый клиентский проект. Не публичный, если есть чувствительные данные.
- [ ] **Vercel** (vercel.com) — подключи GitHub, автодеплой любого HTML/Next.js.
- [ ] **Namecheap / Cloudflare** — домены и A-записи.
- [ ] **Мобильная проверка перед пушем:**
  - `preview_resize` на 375×812 (iPhone)
  - `document.body.scrollWidth === 375` (нет горизонтального скролла)
  - Скриншот hero + main + footer глазами.
- [ ] **Конверсионный аудит** перед отдачей клиенту (15 пунктов: CTA, цифры, trust signals, формы, мобильный).
- [ ] **CLAUDE.md проекта** с инструкцией деплоя (токен, репо, команда push).

**Чек:** «Сделай клиенту Х лендинг по проблеме Y, запушь» — через час работающий URL.

## 🔒 Безопасность (сквозное правило на всех уровнях)

**Правило №1:** Claude видит всё, что ты ему даёшь. Что ушло в контекст — считай публичным.

- [ ] **Секреты — только через `.env`** и `export`. Никогда не вставляй API-ключи в CLAUDE.md, в код, в чат. В `.gitignore` добавь: `.env`, `*.session`, `secrets/`, `credentials.json`.
- [ ] **GitHub токен** — только с минимальными правами (repo — да, admin — нет). Храни в macOS Keychain (`security add-generic-password`), не в текстовом файле.
- [ ] **Permissions mode** — начинай в `ask` (спрашивает перед каждым действием). `accept-edits` только когда освоился. `bypassPermissions` — никогда без причины.
- [ ] **Разделяй проекты** — клиентские данные в отдельных папках, Claude видит только ту, где запущен (`cd project && claude`).
- [ ] **Git pre-commit hook** на секреты — `gitleaks` или `trufflehog`. Ловит ключи до пуша.
- [ ] **.claude/settings.local.json** — личные permissions (не коммить в репо).
- [ ] **Telegram сессии (.session файлы)** = полный доступ к аккаунту. Храни в зашифрованной папке, не в Git.
- [ ] **MCP токены** — rotate раз в 3 месяца. Если запушил случайно — сразу revoke + новый.
- [ ] **Computer Use + браузеры** — в tier "read" по умолчанию (Claude видит, но не кликает). Не снижай без причины.
- [ ] **Клиентские данные (PII, финансы)** — перед отправкой в Claude подумай: это ОК если завтра утечёт? Нет — анонимизируй.

**Правило №2:** никакой команды «удали X» / `rm -rf` / `git reset --hard` / `push --force` без двойного подтверждения. Claude Code спрашивает — не автокликай.

## L6 — Продвинутое (неделя 6+)

**Цель:** Claude работает параллельно и сам, ты только ставишь задачи.

- [ ] **Cowork mode** (Obsidian плагин) — редактируешь markdown вместе с Claude, он видит весь vault.
- [ ] **Parallel agents** — `Task tool` с несколькими subagents в одном сообщении. Делят работу, собирают сводку.
- [ ] **Scheduled tasks** (cron) — «каждое утро 8:00 → email digest → Telegram».
- [ ] **Memory system** — `~/.claude/projects/[project]/memory/` — персональные правила, которые Claude помнит между сессиями.
- [ ] **Context vault** — 8 папок-зеркал под skills (personal, business, values, advisors, projects, tech, finance, people).
- [ ] **RLM-first подход** — большие файлы не читать целиком, сначала `grep -n` / `wc -l`, потом Read с offset/limit.
- [ ] **Session routing** — файлы `TO_<session>.md` для передачи задач между изолированными Cowork-окнами.

**Чек:** у тебя есть хотя бы одна scheduled task, которая присылает тебе что-то полезное сама.

## 📅 Ежедневный прогресс — как Дима будет следить

Каждый день анализ в чате:
1. **Что сделал вчера** — проверка по чек-листу.
2. **Где застрял** — конкретный совет (команда, ссылка, пример).
3. **Следующий шаг** — 1 задача на сегодня, не больше.

Пиши в чат одну из фраз — и я реагирую:
- «Застрял на [уровень]» → разберу, дам конкретный workaround.
- «Сделал [пункт]» → отмечу, подскажу следующий.
- «Покажи пример» → пришлю фрагмент кода/конфига из сетапа Димы.
