## ❓ Что такое Skill на пальцах

Это папка с `SKILL.md` внутри. В `SKILL.md` — описание когда применять + инструкции что делать. Когда ты пишешь в Claude запрос, совпадающий с триггером skill'а — Claude автоматически загружает этот SKILL.md и следует ему.

**Пример:**
```
~/.claude/skills/meeting-prep/
├── SKILL.md          ← триггеры + инструкции
└── template.md       ← шаблон брифа
```

SKILL.md:
```markdown
---
name: meeting-prep
description: Подготовить бриф перед встречей с клиентом
triggers: ["встреча с", "бриф по", "готовлюсь к звонку"]
---

1. Найди в Gmail последние 10 писем с этим клиентом
2. Найди заметки в Obsidian vault
3. Собери: контекст, открытые вопросы, 3 темы для встречи
4. Положи в `Встречи/YYYY-MM-DD-{client}.md`
```

Ты пишешь: «готовлюсь к звонку с Петровым» → Claude сам находит переписку, заметки, собирает бриф.

## 🏗 Три типа skills — ставим в таком порядке

### 1️⃣ Anthropic Skills (готовые, ставятся одной командой)

Это официальные skills от Anthropic. Для документов и веба.

В Claude Code:
```
/plugin marketplace add anthropics/skills
/plugin install anthropic-skills
```

После этого доступно:
- `docx` — создание/правка Word-документов (таблицы, оглавление, картинки)
- `pdf` — чтение/создание PDF, OCR, заполнение форм, объединение
- `xlsx` — Excel: формулы, чарты, очистка данных
- `pptx` — презентации: слайды, шаблоны, speaker notes
- `perplex` — веб-поиск через Perplexity (~$0.006/запрос)
- `youtube-transcript` — вытянуть транскрипт и summary видео
- `skill-creator` — создавать свои skills (на этом держится всё)

**Проверь:** «сделай Excel с бюджетом на месяц» → должен сам применить `xlsx`.

---

### 2️⃣ Superpowers — для разработки и мышления

Плагин от Obra. Ставится так же через marketplace (ищи `obra/superpowers` или в плагин-каталоге).

Ключевые skills:
- `brainstorming` — перед большой задачей: думаем вслух, генерим варианты, выбираем
- `writing-plans` — перед реализацией: пишем план в markdown, согласуем с тобой, потом код
- `systematic-debugging` — при баге: формулируем гипотезу, тестируем, не копаемся наугад
- `verification-before-completion` — перед закрытием: реально ли работает? прогнали тесты? посмотрели в браузере?
- `dispatching-parallel-agents` — параллельные subagents через Task tool

**Правило применения:** эти skills Claude должен применять САМ, не спрашивая. В твоём CLAUDE.md в проекте пропиши:

```markdown
## Skills — применять автоматически
- Перед написанием кода → superpowers:brainstorming
- Перед реализацией фичи → superpowers:writing-plans
- При баге → superpowers:systematic-debugging
- Перед «готово» → superpowers:verification-before-completion
```

### 3️⃣ Impeccable — для UI и дизайна

Нужен если делаешь сайты / лендинги / клиентские интерфейсы. Ставится через marketplace.

- `polish` — финальная полировка (отступы, типографика, контраст)
- `typeset` — типографика (лесенки заголовков, line-height, шрифтовые пары)
- `arrange` — компоновка секций
- `audit` — проверка качества (чек-лист: контраст, размеры, доступность)
- `animate` — микро-анимации (GSAP, CSS transitions)
- `bolder` — усилить визуал, контраст, смелость
- `distill` — упростить, убрать шум

**Типичная цепочка для лендинга:**
arrange → typeset → animate → audit → polish

## 🎨 Твой первый свой skill — за 20 минут

Задача: свой `/morning-brief` — утренняя сводка из Gmail + Calendar + Telegram.

### Шаг 1. Запустить skill-creator
В Claude Code:
```
используй skill-creator, создай skill morning-brief
```

### Шаг 2. Claude спросит описание — скажи:
> Каждое утро читать непрочитанные письма в Gmail, события в Google Calendar на сегодня, важные сообщения в Telegram. Собрать в markdown-файл `3_ЗАДАЧИ/daily-brief/YYYY-MM-DD.md`. Формат: (1) топ-3 письма, (2) встречи сегодня по часам, (3) кому надо ответить в Telegram.

### Шаг 3. Триггеры — скажи:
> «утренний бриф», «morning brief», «что у меня сегодня», «брифинг»

### Шаг 4. Проверь
Перезапусти Claude Code. Напиши: «что у меня сегодня». Должно сработать.

Если не сработало — файл `SKILL.md` лежит в `~/.claude/skills/morning-brief/`. Открой, почитай что написал skill-creator, поправь триггеры вручную.

## 📐 Правила хороших skills (Дима набил за полгода)

1. **Один skill = одна задача.** Не пихай «утренняя сводка + публикация в Telegram + ответ клиентам» в один skill. Три skill'а.
2. **Триггеры — реальные фразы, которые ты говоришь.** Не «initiate morning routine», а «что у меня сегодня», «бриф», «утром».
3. **SKILL.md должен быть коротким.** Если больше 2 экранов — делай дочерние файлы (`template.md`, `examples/`). Claude сам подтянет.
4. **Пиши в повелительном.** Не «в этом skill я описываю как», а «1. Сделай X. 2. Потом Y.»
5. **Явно укажи куда класть результат.** Claude не телепат — скажи путь файла.

## 🔗 Связь с CLAUDE.md проекта

**CLAUDE.md** = «кто я, что за проект, правила конкретно тут».
**Skills** = «как делать типовую задачу везде».

Типичный баланс:
- В CLAUDE.md: пути к папкам клиента, статус проекта, что нельзя трогать.
- В skill'ах: как готовить бриф, как писать коммит, как проверять сайт перед деплоем.

Когда ты ловишь себя на том, что пишешь Claude одно и то же в третий раз — это сигнал делать skill.
